# 车牌识别项目第八周工作报告

生成日期：2026-07-05

## 一、本周工作目标

本周重点围绕 CRNN + CTC 车牌识别模型进行进一步微调，目标是在已有训练成果基础上提升整牌识别准确率，并通过更长训练周期、更大 batch size 和学习率衰减策略，让模型在 CCPD 车牌裁剪图识别任务上获得更稳定的收敛表现。

本周优化重点包括：

1. 将 CRNN 训练从短周期实验推进到更充分的 80 epoch 微调。
2. 将 batch size 从 32 提升到 64，提高单次迭代的数据利用效率。
3. 使用 StepLR 学习率衰减策略，让训练后期更细致地优化模型参数。
4. 记录 GPU 资源占用情况，为后续继续扩大训练规模提供依据。
5. 汇总微调后的识别准确率，并更新当前模型总结。

## 二、本周训练配置

本周使用的主要训练配置如下：

| 配置项 | 本周配置 | 之前配置 | 说明 |
|---|---:|---:|---|
| Screen 会话 | `plate_train_v2` | - | 用于保持长时间训练任务稳定运行 |
| batch size | 64 | 32 | 提升单批次样本量 |
| epochs | 80 | 20 | 增加训练轮数，进行更充分微调 |
| 学习率衰减 | `StepLR(step=15, gamma=0.5)` | - | 每 15 个 epoch 学习率减半 |
| GPU 显存占用 | 1991MB / 24564MB | - | 显存占用较低，仍有扩展空间 |
| GPU 利用率 | 21% | - | 记录时仍处于数据加载阶段 |

学习率调度策略为：

```text
StepLR(step_size=15, gamma=0.5)
```

也就是每训练 15 个 epoch，将当前学习率衰减为原来的一半。相比固定学习率训练，这种方式可以在前期保持较快收敛速度，在后期通过更小学习率细化模型参数，减少训练震荡。

## 三、本周完成的主要工作

### 1. 完成 CRNN 长周期微调

本周将 CRNN 训练周期从之前的 20 个 epoch 扩展到 80 个 epoch。更长的训练周期让模型有更多机会学习 CCPD 车牌图中的字符排列、形近字符差异和整牌序列规律。

同时，batch size 从 32 提升到 64。在当前 GPU 显存占用约 1991MB / 24564MB 的情况下，训练仍有较多显存余量，说明当前 CRNN 模型和输入尺寸对硬件压力不高，后续还可以继续尝试更大的 batch size 或更多 DataLoader worker 来提高吞吐。

### 2. 引入阶段式学习率衰减

本周训练采用 `StepLR(step=15, gamma=0.5)`。该策略让模型在训练过程中逐步降低学习率：

| epoch 阶段 | 学习率变化 |
|---|---|
| 1-15 | 保持初始学习率 |
| 16-30 | 衰减为初始学习率的 0.5 倍 |
| 31-45 | 衰减为初始学习率的 0.25 倍 |
| 46-60 | 衰减为初始学习率的 0.125 倍 |
| 61-75 | 衰减为初始学习率的 0.0625 倍 |
| 76-80 | 继续使用更低学习率进行收尾优化 |

这种配置适合 CRNN 这类序列识别模型：前期快速学习字符结构和序列对齐，后期通过低学习率提升整牌预测稳定性。

### 3. 记录训练资源使用情况

训练过程中观察到 GPU 状态如下：

| 资源项 | 数值 |
|---|---:|
| 显存占用 | 1991MB / 24564MB |
| GPU 利用率 | 21% |
| 状态说明 | 记录时仍在数据加载阶段 |

从资源占用看，当前瓶颈可能不在显存，而更可能与数据加载、图片预处理或训练管线吞吐有关。后续如果希望进一步缩短训练时间，可以优先检查 DataLoader worker 数量、数据读取路径和预处理效率。

## 四、本周模型结果

本周 CRNN 微调后取得了新的最好结果：

| 指标 | 本周结果 | 之前记录 |
|---|---:|---:|
| 整牌准确率 | 98%+ | 94.43% |
| 字符准确率 | 99.7% | 98.94% |

从结果看，本周微调显著提升了完整车牌字符串的识别能力。整牌准确率从之前约 94.43% 提升到 98% 以上，说明模型不仅单字符分类更准确，而且在 CTC 序列解码后的完整车牌输出上更加稳定。

字符准确率达到 99.7%，说明绝大多数字符已经能够被正确识别。对于车牌识别任务来说，字符准确率的小幅提升会明显放大到整牌准确率上，因为整牌准确率要求每一位字符都完全正确。本周结果表明，CRNN 仍然是当前项目中最可靠的整牌识别模块。

## 五、本周成果总结

本周完成了 CRNN 识别模型的一次关键微调升级。通过将 batch size 提升到 64、训练轮数扩展到 80，并加入 `StepLR(step=15, gamma=0.5)` 学习率衰减，模型在车牌识别任务上取得了新的最好表现：整牌准确率达到 98% 以上，字符准确率达到 99.7%。

这次结果进一步验证了 CRNN + CTC 对完整车牌字符串识别的优势。当前项目主流程仍建议采用：

```text
YOLO 车牌检测 -> 裁剪车牌区域 -> CRNN + CTC 整牌识别
```

其中 YOLO 负责定位车牌，CRNN 负责高精度识别车牌号。本周微调后，识别模块已经成为系统中较稳定的一环，后续优化重点可以更多转向端到端真实场景测试、检测框裁切质量、透视矫正和数据加载效率。
